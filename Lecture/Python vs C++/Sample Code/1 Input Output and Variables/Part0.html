General Notes

Python cares about whitespace, while C++ does not. You don't NEED to indent in C++, but it's still a good way to keep your code clean and readable.

Program Structure

Python scripts will just execute immediately from the first line.

C++ requires an entry point - main(). main() is a function, and unless you write additional functions that are called, everything is run from main.

(Of course, you should strive to add classes & functions to you program so you don't have one big program INSIDE the main function!)

An empty C++ program always begins with: 

int main()
{
	return 0;
}

With C++, commands end with a semicolon ; except if you're opening a block of code, like an if statement or while loop.

If you don't end a line of code in C++ with a semicolon, C++ will assume that the line of code will continue on the very next line. We will see this later when we're outputting text to the screen.

Importing/Including

In Python, we can import libraries using: 

import [name]
# OR
from [name] import [item]

In C++, we use

#include <[name]>

In our C++ program, we must import the libraries for input/output to the console, and the string library. 

#include <iostream>	// I/O Stream library
#include <string>	// String library

(Notice that a comment begins with //

Additionally, we can add 

#include <iostream>	// I/O Stream library
#include <string>	// String library
using namespace std;

after our #include. This will make it so we don't have to specify namespace every time. 

// Output without "using namespace std":
std::cout << "Blah" << std::endl;
	
// Output with "using namespace std":
cout << "Blah" << endl;

Finally, notice that the using namespace std; line ends with a semicolon, but #include <iostream> does not.
When you see a pound sign (#), this is a preprocessor comamnd. These aren't actual commands inside the program, the compiler will take these commands and handle them appropriately.

When you #include a library or another source file, the compiler essentially copy-and-pastes that entire file into your existing C++ program.
